# 二次审计报告｜AStock Source Router V0.1

## 一、审计结论

当前版本通过“可开源基础设施雏形”审计，但仍处于 V0.1 内测阶段。它已经具备核心路由、fallback、质量检查、熔断、审计日志和测试桩；真实数据源适配器需要在安装对应 optional dependencies 的联网环境中二次验收。

## 二、已通过项

| 审计项 | 结论 | 说明 |
|---|---|---|
| 项目定位 | 通过 | 本地多源行情路由器，不是行情数据服务 |
| 免费免注册边界 | 通过 | 不接 token、cookie、券商账号、登录态 |
| 盘后/实时分层 | 通过 | daily_kline、trade_calendar 与 realtime_quotes 分离 |
| 字段级路由 | 通过 | SourcePolicy 按 feature 选择数据源 |
| 自动切源 | 通过 | 单源失败后继续尝试下一源 |
| 质量检查 | 通过 | 空表、缺字段会判定失败 |
| 熔断器 | 通过 | 连续失败后进入冷却期 |
| 健康评分 | 通过 | 记录成功率、失败率、P95 延迟、缺字段、空数据 |
| 审计日志 | 通过 | JSONL + SQLite 双写 |
| UTF-8 约束 | 通过 | Windows bat 执行 chcp 65001；文件统一 UTF-8 |
| 单元测试 | 通过 | 本地 9 个 pytest 全部通过 |

## 三、仍需注意项

1. 不要宣传为“永久免费商用行情源”。准确说法是：本地行情源路由器，依赖第三方公开源可用性。
2. 不要复制第三方库核心代码，只调用公开 API。
3. 不要内置 cookie、token、账户、券商登录态。
4. 不要把本项目部署成公共行情 API 服务，避免数据再分发风险。
5. efinance、mootdx、pytdx、Ashare 等适配器必须保留 optional 状态，不作为默认强依赖。
6. 真实数据源字段会随上游变动，需要建立每日/每周字段审计测试。
7. 接入 S0 前，必须通过同一交易日多源交叉校验，不能只看接口是否返回。
8. 盘中数据与盘后数据必须记录请求时间、目标日期、数据日期，防止未来函数污染。

## 五、禁止事项

- 禁止输出买卖建议、策略信号、仓位计划。
- 禁止自动交易、自动下单、券商账户接入。
- 禁止绕过源站限制或高频请求。
- 禁止把第三方数据缓存后对外售卖或再分发。
